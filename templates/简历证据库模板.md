## 三、单段经历证据卡

### 经历名称：____

| 维度 | 记录 |
| --- | --- |
| 背景 | 这个项目或任务为什么存在？当时的问题是什么？ |
| 目标 | 你负责解决什么具体问题？成功标准是什么？ |
| 你的职责 | 哪些部分是你独立完成，哪些是协作完成？ |
| 技术方案 | 使用了哪些技术？为什么这么选？有没有替代方案？ |
| 难点 | 最难的问题是什么？难在哪里？ |
| 过程证据 | 代码链接、截图、文档、PR、测试记录、上线记录 |
| 结果 | 性能、效率、成本、稳定性、用户反馈等量化结果 |
| 复盘 | 如果重做一次，你会保留什么、改掉什么？ |

## 七、自检清单

- [ ] 每条项目经历都能说清楚业务背景。
- [ ] 每个技术点都有使用原因，而不是只写技术名词。
- [ ] 至少准备 1 个失败复盘或取舍案例。
- [ ] 量化结果有来源，不编造数据。
- [ ] 简历 bullet 不超过两行，动词具体，结果清楚。
