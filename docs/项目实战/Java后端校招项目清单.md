# Java 后端校招项目清单：哪些项目值得写进简历

Java 后端校招项目不需要追求“看起来很大”，而要能证明你具备后端工程能力：接口设计、数据库设计、缓存、消息队列、并发控制、权限、安全、监控和问题排查。

## 一、项目选择原则

| 原则 | 说明 |
| --- | --- |
| 场景真实 | 能说清服务谁、解决什么问题 |
| 难点明确 | 至少包含一个可深挖技术点 |
| 可验证 | 能做压测、日志、指标或测试 |
| 可扩展 | 能讨论流量扩大、数据增长、异常兜底 |
| 可独立讲解 | 你本人能讲清代码和设计 |

不要为了“显得高级”硬写微服务、分布式、Kubernetes。面试官更关心你是否真正理解。

## 二、推荐项目清单

| 项目 | 适合程度 | 能体现什么 | 风险 |
| --- | --- | --- | --- |
| 秒杀系统 | 高 | 高并发、缓存、限流、库存一致性、异步削峰 | 容易被问深，需要理解并发和一致性 |
| 短链系统 | 高 | 编码算法、缓存、数据库、跳转性能、统计 | 业务简单，但可讲出系统设计 |
| IM/聊天室 | 中高 | WebSocket、消息投递、在线状态、离线消息 | 实现复杂度较高 |
| API 网关 | 高 | 鉴权、限流、路由、灰度、可观测 | 需要较强工程抽象能力 |
| 权限系统 | 高 | RBAC、数据权限、审计、组织架构 | 容易写成 CRUD，需要补安全边界 |
| 订单系统 | 中高 | 状态机、事务、幂等、支付回调、补偿 | 需要业务流程设计 |
| 博客/论坛 | 中 | 基础 CRUD、搜索、评论、权限 | 太常见，必须做出亮点 |
| 网盘/文件系统 | 中高 | 分片上传、秒传、存储、权限 | 文件存储细节较多 |
| 监控告警系统 | 高 | 指标采集、规则计算、通知、可观测 | 对工程理解要求高 |

## 三、按岗位选择项目

| 目标岗位 | 优先项目 |
| --- | --- |
| Java 后端开发 | 秒杀、短链、订单、权限、API 网关 |
| 中间件/基础架构 | API 网关、监控告警、配置中心、任务调度 |
| 金融/银行科技岗 | 权限、订单、数据对账、监控告警 |
| AI 应用研发 | RAG、Agent 工具平台、评测平台、知识库 |
| 测试开发 | 接口测试平台、自动化测试平台、质量看板 |

## 四、项目不要只写功能

| 普通功能 | 可面试亮点 |
| --- | --- |
| 用户登录 | Token 刷新、单点登录、权限校验、登录风控 |
| 商品查询 | 索引优化、缓存、搜索、热点数据 |
| 下单 | 幂等、库存一致性、状态机、异步消息 |
| 评论 | 敏感词、风控、分页、消息通知 |
| 文件上传 | 分片上传、秒传、断点续传、权限隔离 |

## 五、推荐组合

### 组合 A：稳妥后端求职

1. 权限系统：证明基础工程能力。
2. 短链系统：证明系统设计和性能意识。
3. 秒杀系统：证明高并发和一致性理解。

### 组合 B：AI 应用研发求职

1. RAG 知识库：证明大模型应用能力。
2. Agent 工具平台：证明工具调用和流程编排。
3. 评测平台：证明工程化和质量意识。

### 组合 C：银行/国企科技岗

1. 权限系统：强调安全、审计和合规。
2. 订单/对账系统：强调状态一致性和异常补偿。
3. 监控告警系统：强调稳定运行和问题排查。

## 六、项目简历写法模板

```text
在【业务场景】下，负责【核心模块】设计与实现，
通过【技术方案】解决【关键问题】，
并使用【压测/日志/测试/指标】验证效果，
最终【量化结果或工程收益】。
```

示例：

```text
负责短链跳转链路设计与性能优化，通过本地缓存 + Redis 缓存降低数据库访问，
并对热点短链增加过期刷新策略，在 10 万级模拟访问下将 P95 跳转耗时控制在 30ms 内。
```
