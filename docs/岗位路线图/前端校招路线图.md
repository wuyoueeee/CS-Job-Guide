# 前端校招路线图

前端校招不只是会写页面。更有竞争力的前端候选人，需要展示组件抽象、状态管理、工程化、性能优化和协作能力。

## 一、能力地图

| 模块 | 要掌握什么 |
| --- | --- |
| HTML / CSS | 布局、响应式、兼容性、可访问性 |
| JavaScript | 原型、闭包、异步、事件循环、模块化 |
| 框架 | Vue / React 至少一个熟练 |
| 工程化 | Vite、Webpack、ESLint、构建优化 |
| 性能 | 首屏、懒加载、缓存、包体积 |
| 协作 | 接口联调、组件文档、代码规范 |

## 二、项目建议

| 项目 | 亮点 |
| --- | --- |
| 后台管理系统 | 权限、路由、表格、表单、状态管理 |
| 组件库 | 组件 API、文档、主题、测试 |
| 可视化看板 | 图表、数据刷新、性能优化 |
| AI 工具前端 | 流式输出、会话管理、错误处理 |

## 三、面试追问

1. 组件如何设计受控和非受控状态？
2. 首屏慢怎么排查？
3. React / Vue 更新机制怎么理解？
4. 如何处理权限路由？
5. 如何保证多人协作下的代码质量？

## 四、简历表达重点

少写“熟悉 Vue / React”，多写你解决了什么具体问题：组件复用、性能优化、权限模型、复杂表单、异常处理、联调效率。
