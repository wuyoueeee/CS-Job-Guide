# 测试开发路线图

测试开发的核心不是“点点页面”，而是用工程化方式提升质量和效率。校招准备时，要把自动化、接口测试、质量平台和问题定位讲清楚。

## 一、能力地图

| 模块 | 要掌握什么 |
| --- | --- |
| 编程基础 | Java / Python 至少一门能写工具和脚本 |
| 测试基础 | 测试用例设计、边界值、等价类、缺陷生命周期 |
| 接口测试 | HTTP、Postman、接口自动化、鉴权 |
| 自动化测试 | UI 自动化、接口自动化、数据驱动 |
| 数据库与 Linux | SQL、日志排查、常用命令 |
| 质量平台 | 测试报告、任务调度、覆盖率、告警 |

## 二、项目建议

| 项目 | 亮点 |
| --- | --- |
| 接口自动化测试平台 | 用例管理、执行、报告、环境配置 |
| Web UI 自动化框架 | Page Object、截图、失败重试 |
| Mock 平台 | 依赖隔离、异常场景模拟 |
| 质量看板 | 缺陷趋势、通过率、覆盖率 |

## 三、面试追问

1. 如何设计登录接口测试用例？
2. 自动化测试不稳定怎么处理？
3. 如何区分前端、后端、数据问题？
4. 测试平台如何支持多环境？
5. 你如何衡量测试有效性？

## 四、推荐补强

- 增加一个自动化测试平台项目。
- 为每个项目准备 3 个真实缺陷定位案例。
- 学会用日志、SQL 和抓包定位问题。
