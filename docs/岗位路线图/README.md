岗位路线图用于回答：这个方向要学什么、做什么项目、简历怎么写、面试会怎么问。

| 岗位方向 | 路线图 | 核心交付物 |

| Java 后端 | [Java 后端校招路线图](./Java后端校招路线图.md) | 后端项目、基础知识、系统设计表达 |
| AI 应用研发 | [AI 应用研发路线图](./AI应用研发路线图.md) | RAG / Agent 作品集、评测报告 |
| 银行 / 国企科技岗 | [银行国企科技岗路线图](./银行国企科技岗路线图.md) | 稳定性叙事、笔试准备、签约风险清单 |

| 前端 | [前端校招路线图](./前端校招路线图.md) | 组件化项目、工程化、性能优化案例 |

## 使用方式

1. 先选择一个主投方向，不要同时平均准备多个方向。
2. 按路线图完成最低交付物，再扩展技术广度。
3. 每两周用投递和面试反馈校正路线。
4. 简历版本要和岗位方向匹配，避免一份简历投所有岗位。
