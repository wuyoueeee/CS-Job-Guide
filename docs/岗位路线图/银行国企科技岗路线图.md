# 银行国企科技岗路线图

银行和国企科技岗不是“低配互联网”。它们更看重稳定性、合规意识、业务理解、沟通表达和笔试通过率。

## 一、岗位能力重点

| 能力 | 重点 | 准备方式 |
| --- | --- | --- |
| 基础技术 | Java、数据库、网络、操作系统 | 高频题 + 笔试训练 |
| 业务理解 | 支付、账户、风控、报表、数据治理 | 了解金融科技场景 |
| 稳定性表达 | 为什么选择银行/国企 | 准备长期规划和动机 |
| 合规意识 | 数据安全、权限、审计 | 项目中体现风控思维 |
| 笔试能力 | 行测、英语、技术选择题 | 提前刷题 |

## 二、简历版本调整

| 互联网简历常见写法 | 银行 / 国企版本建议 |
| --- | --- |
| 强调高并发、复杂架构 | 同时强调稳定性、可维护性、数据准确性 |
| 强调快速迭代 | 补充测试、审计、权限和风险控制 |
| 技术词很多 | 增加业务背景和团队协作 |
| 项目结果夸张 | 使用保守、可验证的数据表达 |

## 三、面试高频问题

1. 为什么选择银行 / 国企科技岗？
2. 为什么不去互联网？
3. 你如何看待稳定性和技术成长的关系？
4. 项目里如何保证数据一致性？
5. 如何处理权限和敏感数据？
6. 能否接受轮岗、分配部门或城市调整？
7. 你未来三到五年的职业规划是什么？

## 四、准备节奏

| 阶段 | 任务 |
| --- | --- |
| 投递前 | 准备银行/国企版简历和自我介绍 |
| 笔试前 | 刷行测、英语、技术选择题 |
| 技术面前 | 准备项目稳定性、权限、数据一致性表达 |
| HR 面前 | 准备动机、城市、稳定性、家庭因素 |
| offer 前 | 核实岗位、地点、合同主体和违约条款 |
