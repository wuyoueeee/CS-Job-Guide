# Java 后端校招路线图

Java 后端适合愿意长期打磨工程能力、系统基础和业务建模能力的同学。校招阶段不要追求“什么都学”，优先形成一条能投递、能面试、能讲项目的主线。

## 一、能力地图

| 模块 | 必须掌握 | 面试会怎么问 |
| --- | --- | --- |
| Java 基础 | 集合、并发、异常、IO、泛型 | HashMap、线程池、锁、JVM 对象模型 |
| JVM | 内存区域、GC、类加载、问题排查 | OOM 怎么定位、GC 日志怎么看 |
| MySQL | 索引、事务、锁、SQL 优化 | 慢查询、隔离级别、索引失效 |
| Redis | 数据结构、缓存一致性、分布式锁 | 缓存穿透、击穿、雪崩 |
| Spring | IoC、AOP、事务、自动配置 | 事务失效、Bean 生命周期 |
| 工程能力 | Git、Linux、日志、监控、测试 | 线上问题怎么排查 |
| 系统设计 | 限流、缓存、异步、分库分表 | 秒杀、短链、IM、网关 |

## 二、项目要求

最低准备 1 个主项目和 1 个备选项目。

| 项目类型 | 适合展示 | 推荐材料 |
| --- | --- | --- |
| 秒杀系统 | 高并发、库存一致性、限流降级 | 架构图、压测结果、失败处理 |
| 短链系统 | 编码、重定向、统计、缓存 | 数据模型、热点 Key、扩容方案 |
| IM 系统 | WebSocket、消息投递、在线状态 | 消息可靠性、离线消息、ACK |
| API 网关 | 鉴权、路由、限流、灰度 | 过滤器链、配置中心、可观测 |
| 权限系统 | RBAC、数据权限、审计 | 权限模型、缓存、越权防护 |

参考：[项目实战专题](../项目实战/README.md)

## 三、简历关键词

| 低质量表达 | 更好的表达 |
| --- | --- |
| 使用 Spring Boot 开发系统 | 基于 Spring Boot 设计订单模块，完成接口幂等、库存校验和异常回滚 |
| 使用 Redis 做缓存 | 使用 Redis 缓存热点商品信息，结合过期时间和主动更新降低数据库访问压力 |
| 熟悉 MySQL | 针对订单查询慢 SQL 增加联合索引，并通过 explain 验证扫描行数变化 |
| 使用 MQ | 将耗时任务异步化，处理消息重复消费和失败重试 |

## 四、面试追问

1. 你项目里最复杂的表结构是什么，为什么这样设计？
2. 如果接口突然变慢，你会按什么顺序排查？
3. Redis 和数据库不一致怎么办？
4. 事务为什么会失效？你的项目里怎么避免？
5. 项目流量扩大 10 倍，第一步改哪里？
6. 你做过什么压测？指标是多少？
7. 如果重做项目，你会删掉什么、保留什么？

## 五、30 天准备建议

| 阶段 | 目标 |
| --- | --- |
| 第 1 周 | 整理 Java、MySQL、Redis 高频问题 |
| 第 2 周 | 补主项目架构图、压测和失败案例 |
| 第 3 周 | 刷算法和系统设计题 |
| 第 4 周 | 模拟面试、改简历、批量投递 |
