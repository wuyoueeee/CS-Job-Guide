# 投稿说明

欢迎投稿面经、项目拆解、岗位资料和求职模板。为了让内容长期可维护，请尽量按统一格式提交。

## 一、投稿类型

| 类型 | 放置位置 | 要求 |
| --- | --- | --- |
| 真实面经 | `docs/面经/` | 时间线、问题、项目追问、复盘 |
| 项目拆解 | `docs/项目实战/` | 架构图、亮点、追问、简历写法 |
| 岗位路线 | `docs/岗位路线图/` | 能力地图、项目要求、面试重点 |
| 模板工具 | `templates/` | 可直接复制使用 |
| 纠错补充 | 原文件位置 | 说明问题和依据 |

## 二、投稿前检查

- [ ] 已隐藏个人隐私。
- [ ] 没有公司内部截图、未公开资料或保密内容。
- [ ] 信息来源可说明，涉及招聘政策请标注日期。
- [ ] 内容不是广告、引流或保 offer 承诺。
- [ ] 建议具体可执行，不只给情绪化结论。

## 三、推荐方式

1. 小修改：直接提 Pull Request。
2. 面经投稿：使用 GitHub Issue 的“投稿面经”模板。
3. 资料建议：使用“补充资料”模板。
4. 错误反馈：使用“纠错建议”模板。
