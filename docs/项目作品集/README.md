# 项目作品集模板

项目作品集的目标不是“代码仓库看起来很满”，而是让面试官快速判断：你做过什么、为什么这样做、遇到什么问题、如何验证效果。

## 一、作品集最小结构

```text
project-name/
  README.md
  docs/
    architecture.md
    interview-notes.md
    tradeoffs.md
  assets/
    architecture.png
    demo.gif
  src/
```

| [Java 后端项目 README 模板](./Java后端项目README模板.md) | 秒杀、短链、IM、网关、权限系统 |

## 三、作品集评分标准

| 维度 | 低分表现 | 高分表现 |
| --- | --- | --- |
| 背景 | 只写技术栈 | 写清业务问题和目标 |
| 架构 | 没有图 | 有模块图、数据流和关键链路 |
| 亮点 | 堆名词 | 有取舍、有指标、有失败案例 |
| 可运行 | 无说明 | 有启动步骤、配置说明和截图 |
| 可面试 | 只给代码 | 有追问清单和回答要点 |

## 四、面试前检查

- [ ] README 第一屏能说明项目价值。
- [ ] 架构图能解释核心链路。
- [ ] 至少有 3 个技术取舍。
- [ ] 至少有 1 个失败复盘。
- [ ] 至少有 1 个性能、质量或效果指标。
