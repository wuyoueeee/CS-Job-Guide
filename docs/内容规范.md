# 内容规范

本项目的内容目标是：真实、清晰、可执行、可维护。

## 一、写作原则

| 原则 | 说明 |
| --- | --- |
| 先场景后结论 | 说明这个建议适用于谁、什么阶段 |
| 先证据后判断 | 能给数据、案例、来源就不要只写观点 |
| 先交付物后资料 | 每篇文章最好告诉读者产出什么 |
| 避免绝对化 | 求职建议受学校、城市、行业周期影响 |
| 保持可复盘 | 给清单、模板、问题表，而不是只讲道理 |

## 二、文章基本结构

```markdown
# 标题

开头：这篇文章解决什么问题，适合谁。

## 一、核心结论

## 二、具体方法

## 三、案例或模板

## 四、常见误区

## 五、行动清单

## 延伸阅读
```

## 三、事实和来源

涉及以下内容时，应尽量标注来源和日期：

- 企业招聘时间。
- 岗位要求。
- 政策规则。
- 薪资、违约金、签约条款。
- 官方招聘链接。

如果信息不确定，请写明“以官方公告和书面材料为准”。

## 四、不建议出现的内容

- 保 offer、包过、内部渠道收费等承诺。
- 未脱敏的个人隐私。
- 公司内部系统截图或未公开文档。
- 夸大项目结果或编造指标。
- 无来源的薪资和 HC 传闻。
