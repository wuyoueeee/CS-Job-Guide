# 数据结构与算法刷题计划：从零到面试

算法训练的目标不是追求题量，而是建立常见模式识别能力，并能在面试中讲清楚思路、复杂度和边界条件。

## 一、刷题路线

```mermaid
flowchart LR
    A[基础语法] --> B[数组与字符串]
    B --> C[链表与栈队列]
    C --> D[哈希与双指针]
    D --> E[树与递归]
    E --> F[二分与搜索]
    F --> G[动态规划]
    G --> H[模拟面试]
```

## 二、六周计划

| 周次 | 主题 | 训练重点 |
| --- | --- | --- |
| 第 1 周 | 数组、字符串、哈希 | 遍历、计数、去重 |
| 第 2 周 | 链表、栈、队列 | 指针、模拟、单调结构基础 |
| 第 3 周 | 双指针、滑动窗口、二分 | 区间和边界 |
| 第 4 周 | 树、递归、回溯 | 遍历、状态恢复 |
| 第 5 周 | 图、搜索、堆 | BFS、DFS、优先队列 |
| 第 6 周 | 动态规划与综合题 | 状态定义、转移和初始化 |

## 三、每道题都要记录什么

| 字段 | 内容 |
| --- | --- |
| 题目类型 | 数组 / 树 / 动态规划 |
| 第一反应 | 最初想到的方法 |
| 正确思路 | 核心步骤 |
| 复杂度 | 时间与空间复杂度 |
| 易错点 | 边界、空输入、重复元素 |
| 是否重做 | 一周后能否独立完成 |

## 四、面试表达模板

```text
我先确认一下输入范围和边界条件。
最直接的方法是【方法 A】，复杂度为【复杂度】。
如果需要进一步优化，可以利用【关键性质】改为【方法 B】。
实现时需要注意【边界条件】。
```

## 五、避免无效刷题

1. 不要只看答案，要独立写出代码。
2. 不要只记录“会不会”，还要记录为什么错。
3. 不要过度追求偏题难题，先覆盖高频模式。
4. 每周至少安排一次限时模拟。
5. 对错题进行间隔重做。
