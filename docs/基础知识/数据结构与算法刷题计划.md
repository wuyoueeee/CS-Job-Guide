# 数据结构与算法：六周训练与复盘方法

算法训练最容易给人一种虚假的踏实感：每天打卡、题量不断增加，但一到面试，遇到稍微变形的题还是无从下手。

题量当然有价值，但它只是结果。更重要的是建立三个能力：

1. 识别常见问题模式。
2. 从朴素解法推导到更优解法。
3. 在压力下写出边界正确、能解释的代码。

## 一、面试写题的完整节奏

算法面试不是拿到题后沉默十分钟，然后突然提交代码。一个更可靠的节奏是：

```mermaid
flowchart LR
    A["确认输入与边界"] --> B["说出朴素解法"]
    B --> C["分析复杂度"]
    C --> D["寻找可利用的性质"]
    D --> E["口述优化方案"]
    E --> F["编写代码"]
    F --> G["手动测试"]
```

例如，看到“两数之和”时，不要直接默写哈希表：

1. 先说双重循环可以解决，时间复杂度为 `O(n^2)`。
2. 再说明遍历到当前数字时，如果能快速判断目标差值是否出现过，就可以减少重复搜索。
3. 引出哈希表，将时间复杂度优化到期望 `O(n)`。

这个推导过程比答案本身更重要。

## 二、六周路线：每周围绕模式训练

| 周次 | 主题 | 重点模式 | 验收方式 |
| --- | --- | --- | --- |
| 第 1 周 | 数组、字符串、哈希 | 计数、去重、前缀和 | 能从暴力解法推导哈希优化 |
| 第 2 周 | 链表、栈、队列 | 快慢指针、反转、单调结构基础 | 手写链表边界不出错 |
| 第 3 周 | 双指针、滑动窗口、二分 | 区间维护、单调性、边界收缩 | 能解释二分循环不变量 |
| 第 4 周 | 树、递归、回溯 | 前中后序、路径、状态恢复 | 能画递归树并说明退出条件 |
| 第 5 周 | 图、搜索、堆 | BFS、DFS、拓扑、优先队列 | 能区分层序搜索与深度搜索 |
| 第 6 周 | 动态规划与综合题 | 状态定义、转移、初始化、空间优化 | 能从小规模例子推导状态 |

如果基础薄弱，可以把每周延长。不要为了赶进度，在链表边界都没写稳时强行刷困难动态规划。

## 三、一张真正有用的错题卡

很多错题本只记录题目和答案，复习时依然不会。建议至少记录：

```markdown
## 最长无重复子串

- 第一反应：双重循环枚举子串，再检查重复字符。
- 卡住原因：没有意识到左边界只需要向右移动。
- 关键性质：窗口内始终没有重复字符；右指针扩张后，左指针只前进不后退。
- 易错点：更新字符最近位置时，要避免左指针倒退。
- 复杂度：时间 O(n)，空间 O(k)。
- 重做时间：明天、一周后、两周后。
```

最值得记录的是“卡住原因”和“关键性质”。它们能帮你识别下一道变形题。

## 四、二分查找：训练边界感的最好样本

二分常被认为简单，却最能暴露代码是否严谨。

以左闭右闭区间为例：

```java
int left = 0;
int right = nums.length - 1;

while (left <= right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] == target) {
        return mid;
    }
    if (nums[mid] < target) {
        left = mid + 1;
    } else {
        right = mid - 1;
    }
}
return -1;
```

你要能解释：

- 为什么循环条件是 `left <= right`。
- 为什么更新边界时要跨过 `mid`。
- 空数组和单元素数组会发生什么。
- 如果要找第一个大于等于目标值的位置，循环不变量如何变化。

刷题不是记模板，而是理解模板里的每一个边界。

## 五、动态规划：先定义状态，别急着抄转移方程

动态规划最常见的问题不是写不出循环，而是状态定义含糊。

拿爬楼梯举例：

```text
dp[i] 表示到达第 i 级台阶的方法数。
```

有了定义，再讨论：

- 最后一步从哪里来。
- `dp[i]` 与哪些更小问题有关。
- 初始状态如何设置。
- 是否能压缩空间。

如果你不能用一句话明确说明 `dp[i]` 的含义，后面的方程大概率只是照着答案写。

## 六、每周怎么安排

一个可执行的节奏：

| 训练类型 | 建议频率 | 目的 |
| --- | --- | --- |
| 新题 | 每周 12 至 18 题 | 覆盖模式 |
| 错题重做 | 每周 2 次 | 建立长期记忆 |
| 限时模拟 | 每周 1 次 | 练习压力下表达与编码 |
| 口头讲题 | 每周 2 次 | 检查是否真正理解 |

题量可以根据时间调整，但四种训练最好都保留。

### 限时模拟怎么做

给自己 45 分钟：

1. 5 分钟理解题目、确认输入输出。
2. 8 分钟说清朴素解法和优化方向。
3. 20 分钟完成编码。
4. 7 分钟手动测试边界。
5. 5 分钟复盘表达和错误。

## 七、常见无效刷题方式

### 1. 看懂答案就算做过

阅读时的熟悉感，不等于独立解决能力。至少隔一天关掉答案重写一次。

### 2. 只刷新题，不重做错题

不会的题不断增加，已经错过的题却没有转化成能力。错题重做的优先级应该高于盲目追求数量。

### 3. 只写代码，不解释复杂度

面试官需要理解你的判断过程。每道题至少说清时间复杂度、空间复杂度和关键边界。

### 4. 一上来只做困难题

中等难度的高频模式没有稳定掌握时，困难题容易消耗大量时间，又很难形成可迁移的方法。

## 八、自测问题

1. 为什么滑动窗口里的左指针通常只向右移动？
2. 二分查找中，循环区间的定义会如何影响条件和边界更新？
3. BFS 与 DFS 分别适合什么场景？
4. 写回溯时，为什么递归返回前通常要恢复状态？
5. 定义动态规划状态时，什么叫“可由更小问题推导”？

当你能把这些问题讲清楚，再去拓展题量，会更稳。
