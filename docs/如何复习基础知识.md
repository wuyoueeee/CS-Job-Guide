# 如何复习计算机基础知识

校招复习最常见的问题不是资料不够，而是缺少主线：今天看一点 MySQL，明天刷几道算法题，收藏越来越多，却很难判断自己是否真正掌握。

复习的目标不是“看过”，而是能够在面试中使用自己的语言解释，并经得住继续追问。

## 一、建立自己的知识地图

```mermaid
flowchart LR
    A[目标岗位] --> B[知识模块]
    B --> C[高频问题]
    C --> D[主动回答]
    D --> E[错题记录]
    E --> F[间隔复习]
    F --> D
```

以 Java 后端校招为例，可以按照以下顺序组织：

| 优先级 | 模块 | 重点 |
| --- | --- | --- |
| P0 | Java 基础 | 集合、异常、并发、面向对象 |
| P0 | 数据库 | 索引、事务、锁、MVCC、慢查询 |
| P0 | 计算机网络 | TCP、HTTP、HTTPS、常见状态码 |
| P0 | 算法 | 数组、链表、树、哈希、搜索、动态规划 |
| P1 | Redis | 数据结构、持久化、缓存问题 |
| P1 | JVM | 内存区域、GC、类加载、排查思路 |
| P1 | 操作系统 | 进程线程、内存、IO、并发 |
| P2 | 框架与工程 | Spring、Linux、Git、日志、测试 |

## 二、用三层标准判断掌握程度

| 层次 | 检查方式 |
| --- | --- |
| 能解释 | 能用自己的语言回答“是什么” |
| 能对比 | 能回答“为什么这样设计”“与另一种方案有什么区别” |
| 能应用 | 能结合项目、代码或故障场景说明如何使用 |

例如，学习索引时不要停留在“索引可以提高查询效率”。继续追问：

1. 为什么常见索引结构是 B+ 树？
2. 什么是聚簇索引、二级索引和回表？
3. 哪些 SQL 可能导致索引失效？
4. 如何使用执行计划验证自己的判断？

## 三、每周安排一次闭环

| 环节 | 建议动作 | 产出 |
| --- | --- | --- |
| 学习 | 阅读可靠资料并整理框架 | 一页笔记 |
| 练习 | 回答 10 道面试题 | 错题列表 |
| 应用 | 结合项目或代码举例 | 一个真实场景 |
| 模拟 | 进行 20 分钟口头问答 | 表达薄弱点 |
| 复盘 | 调整下周计划 | 三个优先事项 |

## 四、避免无效努力

1. 不要同时铺开太多主题，每周设置 2 至 3 个核心目标。
2. 不要只背标准答案，尝试自己讲一遍。
3. 不要沉迷收集资料，优先消化已有内容。
4. 不要忽略错题，反复答错的问题最值得整理。
5. 不要把计划排满，至少保留 20% 缓冲时间。

## 行动清单

- [ ] 根据目标岗位画出一张知识地图。
- [ ] 为每个主题准备“定义、对比、应用”三层问题。
- [ ] 每周完成一次模拟面试和错题复盘。
- [ ] 一周后重新回答薄弱问题，确认是否真正掌握。
