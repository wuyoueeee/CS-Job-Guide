# 按人群使用路线

这个页面解决一个问题：不同阶段的人，不应该用同一套阅读顺序。先找到你当前的位置，再按路线交付材料。

## 一、快速选择

| 当前身份 | 最优先目标 | 推荐路线 |
| --- | --- | --- |
| 大一 / 大二 | 建立基础和项目意识 | 基础知识 → 小项目 → 简历证据库 |
| 大三找实习 | 尽快形成可投递材料 | 简历 → 项目讲解 → 投递管理 |
| 秋招冲刺 | 提高面试转化率 | 投递漏斗 → 项目深挖 → 面试复盘 |
| 双非 / 非科班 | 用项目和表达弥补信任差 | 证据库 → 项目作品集 → 模拟面试 |
| 银行 / 国企科技岗 | 稳定性叙事和笔试准备 | 银行国企指南 → 简历版本 → HR 问题 |
| AI 应用研发 | 做出可展示作品集 | AI 路线 → RAG / Agent 项目 → 评测复盘 |
| 已拿 offer | 降低决策和签约风险 | offer 对比 → 签约避坑 → 第一年生存 |

## 二、大一 / 大二路线

目标不是过早海投，而是形成后续可复用的能力资产。

| 阶段 | 要做什么 | 交付物 |
| --- | --- | --- |
| 第 1 阶段 | 学语言、数据结构、数据库、计算机网络 | 一份基础知识地图 |
| 第 2 阶段 | 做 1 个能跑通的后端或 AI 小项目 | GitHub README 和截图 |
| 第 3 阶段 | 记录课程设计、竞赛、开源尝试 | 简历证据库初版 |
| 第 4 阶段 | 了解岗位差异 | 目标岗位优先级 |

## 三、大三实习路线

目标是尽快完成第一轮投递闭环：有简历、有项目、有投递表、有复盘。

| 周期 | 动作 | 交付物 |
| --- | --- | --- |
| 第 1 周 | 重写简历和项目经历 | 一页简历 + 简历证据库 |
| 第 2 周 | 梳理主项目讲解稿 | 3 分钟讲解 + 10 个追问 |
| 第 3 周 | 开始投递并记录数据 | 投递管理表 |
| 第 4 周 | 面试复盘并改简历 | 错题本 + 简历 v2 |

## 四、秋招冲刺路线

秋招不是每天随机刷题和投递，而是按漏斗数据调整。

| 卡点 | 优先动作 | 参考材料 |
| --- | --- | --- |
| 投递无反馈 | 重写简历和岗位关键词 | [简历改造案例库](../简历/简历改造案例库.md) |
| 笔试不过 | 集中补算法和基础题 | [算法与代码面试准备指南](../面试攻略/算法与代码面试准备指南.md) |
| 一面挂 | 补基础表达和项目入口 | [面试官视角](../面试攻略/面试官视角：技术面试的评估维度.md) |

## 五、AI 应用研发路线

AI 应用研发不只看“会调用模型”，更看工程闭环。

| 能力 | 最低交付物 |
| --- | --- |
| Prompt 与业务建模 | 任务拆解、提示词版本、失败样例 |
| RAG | 文档解析、切分、召回、重排、引用、评测 |
| Agent / Tool Calling | 工具权限、确认机制、失败处理 |
| 评测与观测 | 测试集、指标、日志、bad case 复盘 |
| 作品集表达 | README、架构图、Demo、面试追问 |
