# 贡献指南

感谢你愿意参与完善 CS Job Guide。本项目更看重真实经验、可执行建议和长期可维护性，而不是堆叠资料链接。

## 可以贡献什么

| 类型 | 示例 |
| --- | --- |
| 面经投稿 | 公司、岗位、时间线、面试问题、项目追问、复盘建议 |
| 内容补充 | 岗位路线、项目拆解、笔试经验、签约避坑、银行/国企科技岗资料 |
| 纠错建议 | 链接失效、错别字、事实错误、表达不准确、目录缺失 |
| 模板改进 | 简历证据库、投递管理表、面试复盘、offer 对比表 |
| 项目实战 | 架构图、技术亮点、面试追问、简历写法 |

## 内容质量标准

1. 信息尽量来自真实经历、官方资料或可验证来源。
2. 不发布虚假薪资、保 offer、内推收费、灰产资料。
3. 面经投稿请隐藏个人隐私、面试官姓名、内部系统截图和未公开资料。
4. 技术内容要能回答“为什么这样做”，不要只堆技术名词。
5. 项目文章建议包含架构图、核心亮点、面试追问和简历写法。
6. 求职建议要说明适用场景，避免绝对化表达。

## Pull Request 流程

1. Fork 仓库并创建分支。
2. 保持单个 PR 聚焦一个主题，避免同时改很多无关文件。
3. 新增文章请放到对应 `docs/` 子目录；通用模板放到 `templates/`。
4. 修改 README 时，请确认新增链接能打开。
5. 提交 PR 时说明改动内容、适合读者和是否有待核实信息。

## 本地检查

提交前建议至少检查：

```bash
git diff --check
```

如果新增了 Markdown 链接，请手动点开确认路径正确。涉及外部招聘信息、岗位要求、政策规则的内容，请优先引用官方来源并标注日期。

## Issue 使用方式

如果暂时不方便提交 PR，可以使用 Issue 模板：

- 投稿面经：记录真实面试过程和复盘。
- 补充资料：推荐新主题、岗位资料或学习资源。
- 纠错建议：反馈错误链接、错别字、事实错误或排版问题。

维护者会根据内容价值、真实性和可维护性安排合并或后续整理。
